# CLAUDE.md

Guidance for working in this repo. For human-facing docs see `README.md` and
`vignette("codeminer.api")` / `vignette("developer-guide")`.

## What this is

`codeminer.api` is a [plumber](https://www.rplumber.io/) REST wrapper around the
[`codeminer`](https://github.com/codeminer-io/codeminer) R package. Each endpoint
mirrors a `codeminer` function; a matching client function in `R/client.R` calls
it over HTTP. `codeminer` is a GitHub-only dependency (`Remotes:` in
`DESCRIPTION`) ‚Äî install/update it with `pak::pak("codeminer-io/codeminer")`.

## Layout

- `R/endpoints.R` ‚Äî one `add_<name>_endpoint(pr)` per endpoint.
- `R/create_codeminer_api.R` ‚Äî builds the router; every endpoint must be
  registered here.
- `R/endpoint_helpers.R` ‚Äî `codeminer_handler_factory()`, `codeminer_handle()`,
  error formatters, `post_body_api_spec()` (OpenAPI post-processor).
- `R/client.R` ‚Äî exported client functions mirroring the endpoints.
- `R/client-helpers.R` ‚Äî `api_request()` (the shared HTTP/parse/error layer).
- `R/auth.R` ‚Äî client-side auth strategies (`auth_current()`, `auth_use()`, ‚Ä¶).
- `R/run_codeminer_api.R` ‚Äî server launcher (foreground + background).

## Adding an endpoint

Read the **actual** `codeminer` function signature first
(`args(codeminer::FN)`) ‚Äî do not trust an issue's sketch. Then:

1. **`R/endpoints.R`** ‚Äî `add_<name>_endpoint(pr)` calling `pr$handle(method,
   path, handler = codeminer_handler_factory(function(...) codeminer::FN(...)))`.
   Declare **explicit formals** on the inner function ‚Äî args do NOT flow through
   automatically; the factory only forwards body/query args whose names match
   the formals. Carry over all user-facing `codeminer` args (drop only internal
   ones like `con` / `call`). Mirror the literal-default convention of sibling
   endpoints (`type = NULL`, `lookup_version = "latest"`, `col_filters =
   "default"`), not `codeminer`'s `getOption(...)` defaults.
2. **Path** ‚Äî mirror the `codeminer` name exactly: SHOUTY for the codelist verbs
   (`/CODES`, `/CHILDREN`), snake_case for snake_case functions
   (`/get_relationship_tree`).
3. **`create_codeminer_api()`** ‚Äî add `pr <- add_<name>_endpoint(pr)`.
4. **`R/client.R`** ‚Äî mirror with a client function. Conventions: POST with a
   JSON body via `api_request(endpoint, body_params, .return_raw, auth)`;
   `auth = auth_current()` as the last formal; use
   `codeminer::collect_codes_input(..., type = type)` for `...`-style code args.
   Roxygen typically `@inherit codeminer::FN` + `@inheritParams DESCRIPTION`.
   Run `devtools::document()` to update `NAMESPACE` + `man/`.
5. **Non-codelist results** ‚Äî `api_request()` coerces `result` to a single tibble
   by default. If the endpoint returns something else (e.g. `list(nodes,
   edges)`), pass a `.parse_result` function to `api_request()`.

## Errors

`codeminer_handle()` wraps every handler. Errors of class `codeminer_error`
(raised by `codeminer_abort()`, including subclasses like
`codeminer_max_tree_codes_exceeded`) ‚Üí **HTTP 422** with the structured
`cli_message` preserved. Any other error ‚Üí **500** + a server-log warning. So
expected, user-facing failures surface cleanly with no extra handler code; the
client (`api_request()`'s `httr2_http` handler) reconstructs the classed CLI
error. `codeminer_warning` / `codeminer_message` conditions are captured and
replayed client-side.

## OpenAPI

The spec is generated by plumber from the handler formals, then
`post_body_api_spec()` rewrites POST query params into a JSON `requestBody`.
Nothing to hand-maintain ‚Äî a new endpoint appears automatically once registered.

## Testing

- `devtools::test()`. Dummy DB is created by `tests/testthat/setup.R` via
  `codeminer::create_dummy_database()`; tests use ICD-10 codes like `J45`,
  `E11`.
- Unit-level: `test-endpoints.R` (registration), `test-create_codeminer_api.R`
  (route count + paths ‚Äî bump the count when adding one), `test-swagger-spec.R`
  (add the path to `post_paths`).
- **Integration gotcha:** `test-integration.R` / `test-client-integration.R`
  spawn a background server via `callr::r_bg`, which loads the **installed**
  `codeminer.api`, not the dev source. New endpoints 404 until you
  `R CMD INSTALL .` (or `devtools::install()`). Cover both a happy path and the
  error path (e.g. assert a `422` + `error$error_type`).

## House rules

- UK English in prose/docs.
- Keep comments terse.
- Run `air format .` before committing; keep `lintr::lint_package()` clean (the
  `add_<long_name>_endpoint` `object_length` lint is a tolerated exception).
- Bump the dev version in `DESCRIPTION` per PR.
- Do **not** reference downstream repos anywhere (commits, comments, PR bodies).
